# ProjectFlow - Windows 项目管理工具设计文档

## 一、产品概述

### 1.1 产品定位

ProjectFlow 是一款面向个人开发者和小型团队的本地优先（Local-First）项目管理工具，采用 Tauri 构建原生 Windows 应用，提供离线可用、数据本地存储、极致性能的项目管理体验。

### 1.2 核心设计理念

- **本地优先**：所有数据默认存储在本地 SQLite，支持可选的云同步
- **极速体验**：Rust 后端 + 前端框架，启动时间 < 1s
- **隐私安全**：敏感项目数据不上传，支持端到端加密
- **Markdown 原生**：深度集成 Markdown 编辑，支持 Wiki 式项目文档

#### 关键表结构（简化版）

```sql
-- 项目表
CREATE TABLE projects (
    id TEXT PRIMARY KEY,           -- UUID v4
    name TEXT NOT NULL,
    description TEXT,
    status INTEGER NOT NULL,       -- 0:未开始 1:进行中 2:已完成 3:已归档
    start_date TIMESTAMP,
    end_date TIMESTAMP,
    owner_id TEXT,
    settings JSON,                 -- 灵活配置 JSON
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 任务表（支持嵌套，materialized path 或 closure table 可选）
CREATE TABLE tasks (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    parent_id TEXT REFERENCES tasks(id),
    title TEXT NOT NULL,
    description TEXT,
    status INTEGER NOT NULL DEFAULT 0,  -- 工作流状态
    priority INTEGER DEFAULT 1,         -- 0:最低 3:最高
    assignee_id TEXT REFERENCES users(id),
    start_date TIMESTAMP,
    due_date TIMESTAMP,
    estimated_hours REAL,
    actual_hours REAL DEFAULT 0,
    progress REAL DEFAULT 0 CHECK (progress >= 0 AND progress <= 100),
    position INTEGER DEFAULT 0,         -- 同层级排序
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 任务依赖表（支持 FS/SS/FF/SF）
CREATE TABLE task_dependencies (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    depends_on_task_id TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    dependency_type INTEGER DEFAULT 0,  -- 0:FS 1:SS 2:FF 3:SF
    lag_days INTEGER DEFAULT 0,
    UNIQUE(task_id, depends_on_task_id)
);

-- 文档表（支持版本历史）
CREATE TABLE documents (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id) ON DELETE SET NULL,
    title TEXT NOT NULL,
    content TEXT,                      -- Markdown 内容
    file_path TEXT UNIQUE,             -- 本地文件路径
    current_version INTEGER DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 文档版本历史
CREATE TABLE document_versions (
    id TEXT PRIMARY KEY,
    document_id TEXT NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    version INTEGER NOT NULL,
    content TEXT,
    change_summary TEXT,
    created_by TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(document_id, version)
);

-- 全文搜索索引（Tantivy 管理，此处为引用）
CREATE VIRTUAL TABLE document_search USING fts5(
    title,

    content_rowid=rowid,
    tokenize='porter'
```

#### 3.4.2 服务层 (Service Layer)

```rust
// src/services/scheduler_service.rs
// 定时任务与提醒服务

pub struct SchedulerService {
    db: Database,
    notifier: NotificationManager,
}

impl SchedulerService {
    // 启动后台调度器
    pub async fn start(&self) {
        // 使用 Tokio 定时检查
        let mut interval = tokio::time::interval(Duration::from_secs(60));
        loop {
            interval.tick().await;
            self.check_deadlines().await;
            self.check_risks().await;
        }
    }

    async fn check_deadlines(&self) {
        let upcoming = self.db.tasks()
            .find_due_within(Duration::hours(24))
            .await;

        for task in upcoming {
            self.notifier.send(Notification::DeadlineApproaching(task));
        }
    }
}
```

```rust
// src/commands/task_commands.rs
// 暴露给前端的命令

#[tauri::command]
async fn create_task(
    app: AppHandle,
    payload: CreateTaskPayload,
) -> Result<Task, CommandError> {
    let db = app.state::<Database>();
    let task = db.tasks().create(payload.into()).await?;

    // 广播实时更新（多窗口同步）
    app.emit_all("task:created", &task)?;

    Ok(task)
}

#[tauri::command]
async fn get_gantt_data(
    app: AppHandle,
    project_id: String,
) -> Result<GanttData, CommandError> {
    let db = app.state::<Database>();
    let tasks = db.tasks().find_for_gantt(&project_id).await?;
    let deps = db.dependencies().find_by_project(&project_id).await?;

    Ok(GanttData { tasks, dependencies: deps })
}
```

#### 3.5.2 数据流设计

```
用户操作 → React Component → Zustand Store → TanStack Query
                                              ↓
                                    Tauri Command (IPC)
                                              ↓
                                    Rust Backend + SQLite
                                              ↓
                                    乐观更新/缓存失效 → UI 更新
```

**关键模式**：

- **乐观更新**：UI 立即响应，后台同步，失败回滚
- **本地缓存**：TanStack Query 缓存所有数据，支持离线浏览
- **实时同步**：Rust 端通过 Tauri Event 推送变更，多窗口同步

#### 3.6.3 风险预警算法

```rust
// 风险评分算法（简化版）
pub fn calculate_risk_score(task: &Task, project: &Project) -> RiskAssessment {
    let mut score = 0.0;
    let mut factors = vec![];

    // 1. 进度偏差 (0-40分)
    if let Some(due) = task.due_date {
        let remaining = due - now();
        let estimated_remaining = task.estimated_hours - task.actual_hours;
        if estimated_remaining > remaining_hours(remaining) {
            let deviation = estimated_remaining / remaining_hours(remaining);
            score += (deviation * 20.0).min(40.0);
            factors.push(RiskFactor::ScheduleDeviation(deviation));
        }
    }

    // 2. 依赖风险 (0-30分)
    let blockers = task.dependencies.iter()
        .filter(|d| d.status != Status::Done)
        .count();
    score += (blockers as f32 * 10.0).min(30.0);

    // 3. 资源冲突 (0-30分)
    if let Some(assignee) = task.assignee {
        let workload = calculate_workload(assignee, task.due_date);
        if workload > 1.5 { // 超负荷 150%
            score += 30.0;
            factors.push(RiskFactor::Overall);
        }
    }

    RiskAssessment {
        score: score.min(100.0),
        level: if score > 70 { High } else if score > 30 { Medium } else { Low },
        factors,
        suggested_actions: generate_suggestions(&factors),
    }
}
```

## Problem

需要为 ProjectFlow 添加 LLM 辅助功能，让用户通过对话方式编写项目文档。

## 需求（已确认）

### 1. 多模型支持
- 使用第三方库实现（通用框架 LangChain Rust）
- 支持多种 LLM 提供商

### 2. 对话功能
- 通过新模块实现（独立于文档编辑）
- 独立菜单项："AI 助手"

### 3. 文档类型
- 支持所有文档类型

### 4. 功能范围
- 文档生成
- 回答关于项目的问题
- 分析现有文档并提出建议

### 5. API Key 管理
- 用户自行配置
- 应用内加密存储（AES）

### 6. 上下文理解
- 自动将当前项目信息注入对话：
  - 项目名称、描述
  - 任务列表
  - 文档结构
  - 文档内容摘要

---

## 技术实现方案（草案）

### 阶段一：基础框架
1. 集成 LangChain Rust
2. 模型适配层设计（支持 OpenAI、Claude 等）
3. API Key 加密存储
4. 设置界面

### 阶段二：对话模块
1. 前端对话界面
2. 消息历史存储（SQLite）
3. 对话上下文管理

### 阶段三：项目上下文
1. 项目信息提取
2. 任务列表生成
3. 文档结构解析
4. 内容摘要生成
5. 自动注入 Prompt

### 阶段四：高级功能
1. 文档分析和建议
2. 多轮对话优化
3. 对话导出

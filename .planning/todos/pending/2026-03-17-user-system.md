## Problem

需要为 ProjectFlow 添加用户体系功能，目前系统缺少用户管理和认证相关功能。

## 需求（已确认）

### 1. 用户体系
- 本地账户 + 权限系统，SQLite 存储
- 角色：管理员（1个）、普通用户

### 2. P2P 联网协作
- P2P 长连接方式
- 手动链接：用户手动输入对方 IP/端口
- 支持离线使用，连网后自动同步

### 3. 管理员选举机制
- 初始管理员 = 第一个注册用户
- 当 P2P 网络中没有管理员时，所有客户端投票选举
- 超过 2/3 人员同意则成为管理员

### 4. 项目管理
- 分片同步：每个客户端只同步自己被授权的项目数据
- 所有项目归管理员管理，由管理员分配
- 用户操作流程：申请建项目 → 管理员审核 → 分配权限 → 操作

### 5. 管理员功能
- 用户管理（新增/删除）
- 权限分配（项目分配、任务分配）
- 项目/文档审核

### 6. 冲突解决
- 手动解决：用户选择保留哪个版本

### 7. 优先级
- 较低，可后续迭代

---

## 技术实现方案（草案）

### 阶段一：基础用户体系
1. 用户注册/登录功能
2. 用户表设计（id, username, password_hash, role, created_at）
3. 权限表设计（user_id, project_id, permission）
4. 登录界面 + 用户管理界面

### 阶段二：P2P 网络层
1. 选型：libp2p (Rust)
2. 实现手动连接（输入 IP:端口）
3. 节点发现和邻居维护
4. 消息协议设计（类型：用户数据、项目数据、任务数据、投票）

### 阶段三：数据同步
1. 分片同步逻辑
2. 离线数据队列
3. 连线后自动同步

### 阶段四：管理员选举
1. 选举状态机
2. 投票协议实现
3. 2/3 多数决

### 阶段五：冲突解决
1. 版本向量记录
2. 冲突检测
3. 手动解决 UI

---

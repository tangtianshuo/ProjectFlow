# ProjectFlow 黑盒测试用例文档

## 测试范围
本测试用例针对 ProjectFlow 项目管理系统的核心功能模块进行黑盒测试。

---

## 测试用例

### 模块一：项目管理

#### TC-PM-001: 创建新项目
| 项目 | 内容 |
|------|------|
| 用例ID | TC-PM-001 |
| 用例名称 | 创建新项目 |
| 测试目的 | 验证用户可以成功创建一个新项目 |
| 前置条件 | 应用已启动，显示项目列表页面 |
| 测试步骤 | 1. 点击"新建项目"按钮<br>2. 填写项目名称<br>3. (可选)填写项目描述<br>4. (可选)填写开始日期和结束日期<br>5. 点击"创建"按钮 |
| 预期结果 | 项目成功创建，显示在项目列表中 |
| 测试优先级 | 高 |

#### TC-PM-002: 启动项目
| 项目 | 内容 |
|------|------|
| 用例ID | TC-PM-002 |
| 用例名称 | 启动项目 |
| 测试目的 | 验证可以将未开始的项目启动为进行中状态 |
| 前置条件 | 存在状态为"未开始"的项目 |
| 测试步骤 | 1. 找到状态为"未开始"的项目<br>2. 点击"开始项目"按钮 |
| 预期结果 | 项目状态变为"进行中"，按钮文字变为"完成项目" |
| 测试优先级 | 高 |

#### TC-PM-003: 完成项目
| 项目 | 内容 |
|------|------|
| 用例ID | TC-PM-003 |
| 用例名称 | 完成项目 |
| 测试目的 | 验证可以将进行中的项目标记为已完成 |
| 前置条件 | 存在状态为"进行中"的项目 |
| 测试步骤 | 1. 找到状态为"进行中"的项目<br>2. 点击"完成项目"按钮 |
| 预期结果 | 项目状态变为"已完成"，按钮文字变为"重新开启" |
| 测试优先级 | 高 |

#### TC-PM-004: 重新开启项目
| 项目 | 内容 |
|------|------|
| 用例ID | TC-PM-004 |
| 用例名称 | 重新开启已完成项目 |

| 前置条件 | 存在状态为"已完成"的项目 |
| 测试步骤 | 1. 找到状态为"已完成"的项目<br>2. 点击"重新开启"按钮 |
| 预期结果 | 项目状态变为"进行中"，按钮文字变为"完成项目" |
| 测试优先级 | 高 |

#### TC-PM-005: 删除项目
| 项目 | 内容 |
|------|------|
| 用例ID | TC-PM-005 |
| 用例名称 | 删除项目 |
| 测试目的 | 验证可以删除项目 |
| 前置条件 | 存在项目 |
| 测试步骤 | 1. 找到要删除的项目<br>2. 点击删除按钮<br>3. 在确认对话框中点击"确定" |
| 预期结果 | 项目从列表中删除 |
| 测试优先级 | 中 |

#### TC-PM-006: 项目进度计算
| 项目 | 内容 |
|------|------|
| 用例ID | TC-PM-006 |
| 用例名称 | 项目进度根据任务完成情况计算 |
| 测试目的 | 验证项目进度根据任务完成百分比自动计算 |
| 前置条件 | 项目中有多个任务，且任务不同状态 |
| 测试步骤 | 1. 查看项目处于列表<br>2. 检查每个项目的进度条 |
| 预期结果 | 进度条显示正确的完成百分比 = (已完成任务数 / 总任务数) * 100% |
| 测试优先级 | 中 |

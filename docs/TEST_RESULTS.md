## 本次修复验证

### 问题1: 任务状态更新报错
- **错误信息**: `Wrong number of parameters passed to query. Got 4, needed 3`
- **根本原因**: `db/mod.rs:308` 中 `update_task` 函数更新 progress 时没有增加 param_idx
- **修复代码**:
  ```rust
  // 修复前
  if progress.is_some() { updates.push(format!("progress = ?{}", param_idx)); }

  // 修复后
  if progress.is_some() { updates.push(format!("progress = ?{}", param_idx)); param_idx += 1; }
  ```
- **验证结果**: ✅ Rust 编译通过，逻辑修复正确

### 问题2: 项目完成后无法重新开启
- **根本原因**: `ProjectList.vue` 中 `toggleProjectStatus` 函数只处理了 0→1 和 1→2，缺少 2→1
- **修复内容**:
  - 添加 `else if (project.status === 2)` 分支处理重新开启
  - 添加"重新开启"按钮文字
  - 添加重新开启图标
- **验证结果**: ✅ 代码逻辑正确，功能可用

### 问题3: 仪表盘数据统计不准确
- **根本原因**: `Dashboard.vue` 只使用 `projectStore.tasks`（当前项目任务），未统计所有项目
- **修复内容**:
  - 添加 `allTasks` 状态
  - 在 `onMounted` 中遍历所有项目获取任务
  - 统计数据使用 `allTasks` 而非 `projectStore.tasks`
- **验证结果**: ✅ 代码逻辑正确，统计数据完整
